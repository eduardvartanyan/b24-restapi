# Проект "Ремонт ТС": опорные данные

Дата создания: 2026-06-17

## Назначение проекта

Проект нужен для учета и контроля услуги организации ремонта поврежденного транспортного средства через партнерские СТОА.

Клиент получает рекомендацию СТОА, записывается на дефектовку, далее Форсайт контролирует начало ремонта, окончание ремонта, качество ремонта и расчеты с СТОА за привлеченного клиента.

## Рекомендуемые стадии воронки CRM

1. Сделка создана
2. Подбор СТОА
3. Запись на дефектовку
4. Дефектовка назначена
5. Результат дефектовки получен
6. Клиент думает
7. Контроль начала ремонта
8. Ремонт в работе
9. Контроль окончания ремонта
10. Проверка качества
11. Расчеты
12. Успешно завершено
13. Отказ / неуспешно

## Набор полей карточки сделки

### Клиент и источник

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | --- |
| Клиент | Контакт CRM | Да | Основной контакт клиента. |
| Источник обращения | Список | Да | ДТП, офис, входящий звонок, виновник, другое. |
| Связанный проект/сделка | Привязка к CRM | Нет | Для будущей интеграции с проектами оформления ДТП и получения документов по ДТП. |

### Транспортное средство

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | -- |
| Марка ТС | Строка | Да | |
| Модель ТС | Строка | Да | |
| Госномер | Строка | Да | Желательно единый формат ввода. |
| VIN | Строка | Нет | Нужен, если СТОА или эксперты используют VIN в работе. |
| Описание повреждений | Многострочный текст | Да | Краткое описание со слов клиента/менеджера. |
| Фото повреждений | Файл | Нет | Если процесс будет требовать первичной фиксации. |

### СТОА и запись

| Поле | Тип                         | Обязательность | Примечание |
| --- |-----------------------------| --- | --- |
| СТОА | Смарт-процесс "СТОА"        | Да со стадии "Запись на дефектовку" | Лучше хранить СТОА как отдельный справочник/смарт-процесс. |
| Мастер СТОА | Ресурс бронирования         | Да со стадии "Запись на дефектовку" | Для внешних мастеров нужно проверить модель доступа. |
| Адрес СТОА | Строка/автозаполнение       | Нет | Можно подтягивать из карточки СТОА. |
| Дата и время дефектовки | Дата/время или бронирование | Да со стадии "Дефектовка назначена" | Основной кандидат: онлайн-запись/бронирование ресурсов CRM. |
| Статус подтверждения записи | Список                      | Нет | Не отправлено, отправлено, подтверждено, перенесено, отменено. |

### Дефектовка

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | --- |
| Результат дефектовки | Список | Да после дефектовки | Отказался, думает, согласился на ремонт, не приехал, перенес запись. |
| Комментарий мастера | Многострочный текст | Нет | Итоги осмотра, замечания, ориентировочная стоимость. |
| Ориентировочная стоимость ремонта | Деньги | Нет | Если СТОА готова передавать оценку. |

### Ремонт

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | --- |
| Плановая дата начала ремонта | Дата | Нет | Заполняется после согласия клиента. |
| Фактическая дата начала ремонта | Дата | Нет | Фиксирует реальный старт. |
| Плановая дата окончания ремонта | Дата | Нет | Используется для контроля. |
| Фактическая дата окончания ремонта | Дата | Нет | Заполняется по информации от мастера/клиента. |
| Статус ремонта | Список | Нет | Не начат, в работе, приостановлен, завершен. |

### Проверка качества

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | --- |
| Эксперт Форсайта | Пользователь | Нет | Ответственный за проверку качества. |
| Дата проверки качества | Дата/время | Нет | |
| Результат проверки качества | Список | Да на стадии "Проверка качества" | Принято без замечаний, есть замечания, требуется повторная проверка. |
| Комментарий эксперта | Многострочный текст | Нет | Рекомендации клиенту по приемке ТС. |

### Расчеты

| Поле | Тип | Обязательность | Примечание |
| --- | --- | --- | --- |
| Сумма комиссии | Деньги | Да на стадии "Расчеты" | Комиссия Форсайта за привлеченного клиента. |
| Товар/услуга | Товар CRM | Да на стадии "Расчеты" | "Комиссия за ремонт". |
| Счет/оплата от СТОА | Счет/оплата CRM | Да на стадии "Расчеты" | Нужно проверить используемый в портале механизм счетов/оплат. |
| Статус оплаты от СТОА | Список | Да на стадии "Расчеты" | Не выставлено, выставлено, оплачено, просрочено. |
