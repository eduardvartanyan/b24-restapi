# Проект "Ремонт ТС": открытые вопросы

Дата создания: 2026-06-17
Дата актуализации: 2026-06-29

Этот файл хранит вопросы, которые нужно уточнить перед финальной настройкой CRM-воронки в облачном Битрикс24. Вопросы синхронизированы с текущими файлами `projects/remont-ts-tz.md` и `projects/remont-ts-reference.md`.

## 1. Битрикс24 и онлайн-запись

| Вопрос | Статус | Комментарий |
| --- | --- | --- |
| Какой тариф Битрикс24 используется сейчас? | Закрыт | Облако, профессиональный тариф. |
| Доступна ли функция "Онлайн-запись" в текущем портале? | Закрыт | Да, доступна. |
| Можно ли завести мастеров СТОА как ресурсы бронирования или специалистов? | Закрыт | Да, можно. В карточке сделки поле "Мастер СТОА" задано как ресурс бронирования. |
| Можно ли менеджеру Форсайта вручную создавать онлайн-запись из CRM-сделки? | Закрыт | Да, можно. Клиент сам не должен пользоваться публичной формой. |
| Можно ли хранить запись онлайн-записи в карточке сделки? | Закрыт | Да, можно. |
| Можно ли запускать роботов от создания или изменения онлайн-записи? | Закрыт | Можно запускать от создания онлайн-записи. От изменения запуск не подтвержден. |

## 2. СТОА и мастера

| Вопрос | Статус | Комментарий |
| --- | --- | --- |
| Какие поля должны быть в смарт-процессе "СТОА"? | Открыт | Минимум: название, адрес, телефоны, график работы, список мастеров, условия комиссии, комментарии. |
| Сколько СТОА планируется подключить на первом этапе? | Открыт | В ТЗ для пилота указано 1-2 тестовые СТОА. |
| Сколько мастеров будет у каждой СТОА? | Открыт | В ТЗ для пилота указано 2-3 тестовых мастера. |
| У каждого мастера должен быть отдельный график приема? | Закрыт | Да, у каждого мастера должен быть отдельный график приема. |
| Есть ли у СТОА стабильные графики приема на дефектовку? | Открыт | Влияет на настройку доступных окон в онлайн-записи. |

## 3. Задачи мастерам

| Вопрос | Статус | Комментарий |
| --- | --- | --- |
| Как именно мастер получает ссылку на автоматически созданную задачу? | Закрыт | Через каскадную рассылку: Макс -> WhatsApp -> Telegram -> SMS. Рассылка уже настроена в системе. |
| Может ли мастер открыть задачу и оставить комментарий без доступа к лишней CRM? | Закрыт | Да, может. |
| Какие поля задачи мастер должен видеть? | Открыт | Минимум: ТС, госномер, клиент, телефон, дата/время, требуемый результат. |
| Как результат из комментария к задаче будет переноситься в поле "Результат дефектовки"? | Открыт | Нужно решить: вручную ответственным или автоматизацией. |
| Нужна ли стандартизированная форма комментария мастера? | Открыт | Например: результат, комментарий, ориентировочная стоимость ремонта. |

| --- | --- | --- |
