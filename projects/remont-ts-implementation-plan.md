# Проект "Ремонт ТС": план внедрения

Дата создания: 2026-06-17

Цель плана: разложить настройку CRM-воронки в облачном Битрикс24 на управляемые этапы.

## 1. Подготовка

1. Уточнить тариф Битрикс24 и доступность нужных функций.
2. Проверить наличие "Онлайн-записи" в портале.
3. Проверить текущую модель счетов, оплат и товаров в CRM.
4. Согласовать список участников пилота: менеджеры, эксперты, СТОА, мастера.
5. Согласовать минимальный набор персональных данных, которые можно передавать мастерам СТОА.

Результат этапа: принято решение, можно ли использовать онлайн-запись и внешних пользователей в целевой модели.

## 2. Структура CRM

1. Создать тестовую воронку "Ремонт ТС".
2. Создать стадии воронки по файлу `projects/remont-ts-reference.md`.
3. Создать пользовательские поля сделки.
4. Настроить обязательность ключевых полей по стадиям.
5. Создать товар/услугу "Комиссия за ремонт".

Результат этапа: менеджер может создать тестовую сделку и провести ее вручную по стадиям.

## 3. Справочник СТОА

1. Выбрать способ хранения СТОА: список, справочник или смарт-процесс.
2. Создать структуру карточки СТОА.
3. Завести тестовые СТОА.
4. Завести тестовых мастеров.
5. Связать СТОА и мастеров с карточкой сделки.

Результат этапа: менеджер может выбрать СТОА и мастера в сделке.

## 4. Запись и календари

Основной вариант: онлайн-запись/бронирование внутри CRM.

1. Настроить тестовую услугу "Дефектовка ТС".
2. Настроить ресурсы или специалистов: мастера СТОА.
3. Настроить графики приема по мастерам/СТОА.
4. Проверить ручное создание записи менеджером.
5. Проверить отображение занятости мастера.
6. Проверить фиксацию записи в карточке CRM.
7. Проверить уведомления клиенту и мастеру.

Резервный вариант: календарные события и поля сделки.

1. Создать отдельные календари мастеров.
2. Настроить ручное создание события календаря.
3. Зафиксировать в сделке СТОА, мастера, дату и время.
4. Настроить задачу ответственному или мастеру.
5. Проверить риск двойной записи.

Результат этапа: выбран и протестирован рабочий способ записи на дефектовку.

## 5. Права доступа

1. Создать тестового внешнего мастера.
2. Проверить доступ к задачам.
3. Проверить доступ к календарю/записям.
4. Проверить доступ к CRM, ленте, чатам и контактам.
5. Зафиксировать допустимую модель доступа.
6. Если доступы слишком широкие, перейти к модели формы/ссылки без полноценного пользователя Битрикс24.

Результат этапа: подтвержден безопасный способ взаимодействия с мастерами СТОА.

## 6. Роботы и задачи

1. На стадии "Сделка создана" поставить задачу подобрать СТОА.
2. На стадии "Запись на дефектовку" создать запись/задачу мастеру.
3. На стадии "Дефектовка назначена" поставить контроль результата.
4. По результату дефектовки перевести сделку в нужную стадию.
5. На стадии "Клиент думает" поставить задачу повторного контакта.
6. На стадии "Контроль начала ремонта" поставить задачу ответственному.
7. На стадии "Ремонт в работе" поставить задачу контроля окончания.
8. На стадии "Проверка качества" поставить задачу эксперту.
9. На стадии "Расчеты" поставить задачу контроля оплаты.

Результат этапа: сделка сопровождается автоматическими задачами и подсказками ответственным.

## 7. Уведомления

1. Согласовать каналы уведомлений клиента.
2. Настроить подтверждение записи клиенту.
3. Настроить напоминание клиенту о дефектовке.
4. Проверить возможность уведомлений мастера в МАКС.
5. Если МАКС недоступен, выбрать резервный канал.
6. Настроить шаблоны сообщений.

Результат этапа: ключевые участники получают уведомления без ручного дублирования, насколько это возможно в текущей инфраструктуре.

## 8. Отчеты

1. Настроить отчет по количеству сделок и источникам.
2. Настроить отчет по конверсии из дефектовки в ремонт.
3. Настроить отчет по отказам и причинам.
4. Настроить отчет по загрузке СТОА/мастеров, если данные доступны.
5. Настроить отчет по начисленной и оплаченной комиссии.
6. Настроить контроль просроченных оплат.

Результат этапа: руководитель видит воронку, эффективность СТОА и деньги.

## 9. Пилот

1. Провести 3-5 тестовых сделок на тестовых данных.
2. Провести 3-5 реальных сделок с ограниченным числом СТОА.
3. Собрать замечания менеджеров, экспертов и мастеров.
4. Исправить поля, стадии, задачи и уведомления.
5. Согласовать финальный регламент.

Результат этапа: процесс готов к запуску на всех подключенных СТОА.

## 10. Запуск

1. Перенести настройки из тестового режима в рабочий.
2. Обучить менеджеров и экспертов.
3. Подключить мастеров/СТОА.
4. Запустить процесс.
5. Первые 1-2 недели вести ежедневный контроль ошибок и ручных обходов.

Результат этапа: воронка "Ремонт ТС" работает в регулярном режиме.
