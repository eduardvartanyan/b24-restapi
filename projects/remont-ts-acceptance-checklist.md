# Проект "Ремонт ТС": чеклист приемки

Дата создания: 2026-06-17

Этот чеклист используется для проверки, что настройка CRM-воронки в Битрикс24 действительно готова к работе.

## 1. Воронка и карточка сделки

- [ ] Создана отдельная воронка "Ремонт ТС".
- [ ] Созданы все согласованные стадии.
- [ ] Созданы обязательные поля клиента и ТС.
- [ ] Созданы поля СТОА, мастера и даты дефектовки.
- [ ] Созданы поля результата дефектовки.
- [ ] Созданы поля контроля ремонта.
- [ ] Созданы поля проверки качества.
- [ ] Созданы поля расчетов и комиссии.
- [ ] Обязательность полей настроена по стадиям.

## 2. СТОА и мастера

- [ ] Выбран способ хранения СТОА: список, справочник или смарт-процесс.
- [ ] В тестовом контуре заведены СТОА.
- [ ] В тестовом контуре заведены мастера.
- [ ] Сделка связывается с выбранной СТОА.
- [ ] Сделка связывается с выбранным мастером.
- [ ] В карточке СТОА хранится адрес и рабочий график.
- [ ] В карточке СТОА хранится информация по комиссии.

## 3. Онлайн-запись / календари

- [ ] Проверена доступность онлайн-записи на текущем тарифе.
- [ ] Настроена тестовая услуга "Дефектовка ТС".
- [ ] Настроены мастера как специалисты или ресурсы.
- [ ] У каждого мастера есть видимая занятость.
- [ ] Менеджер видит доступные окна мастеров.
- [ ] Менеджер может записать клиента вручную.
- [ ] Запись фиксируется в карточке CRM.
- [ ] Запись не создает конфликтов и двойных бронирований.
- [ ] Если онлайн-запись не подходит, проверен резервный календарный сценарий.

## 4. Задачи и роботы

- [ ] При создании сделки ставится задача подобрать СТОА.
- [ ] После выбора времени создается задача/запись на дефектовку.
- [ ] Мастер или ответственный получает уведомление о дефектовке.
- [ ] После дефектовки фиксируется результат.
- [ ] По результату "отказался" сделка уходит в неуспешную стадию.
- [ ] По результату "думает" создается задача повторного контакта.
- [ ] По результату "согласился" создается задача контроля начала ремонта.
- [ ] На стадии ремонта создается задача контроля окончания.
- [ ] На стадии проверки качества создается задача эксперту.
- [ ] На стадии расчетов создается задача контроля оплаты.

## 5. Права доступа

- [ ] Менеджер Форсайта видит и редактирует свои сделки.
- [ ] Руководитель видит все сделки направления.
- [ ] Эксперт видит сделки/задачи, где он назначен.
- [ ] Внешний мастер не видит лишнюю CRM.
- [ ] Внешний мастер не видит ленту и внутренние чаты.
- [ ] Внешний мастер видит только минимальные данные, необходимые для дефектовки.
- [ ] Если безопасный доступ мастера невозможен, утвержден альтернативный сценарий без полноценного пользователя Битрикс24.

## 6. Уведомления

- [ ] Клиент получает подтверждение записи, если такой сценарий утвержден.
- [ ] Клиент получает напоминание о визите, если такой сценарий утвержден.
- [ ] Мастер получает уведомление о новой записи.
- [ ] Ответственный получает напоминание проверить результат дефектовки.
- [ ] Ответственный получает задачи по началу и окончанию ремонта.
- [ ] Проверена возможность уведомлений в МАКС.
- [ ] Если МАКС не подходит, выбран резервный канал.

## 7. Расчеты

- [ ] Создан товар/услуга "Комиссия за ремонт".
- [ ] В сделке фиксируется сумма комиссии.
- [ ] Понятен способ выставления счета/оплаты СТОА.
- [ ] В сделке фиксируется статус оплаты.
- [ ] Есть контроль просроченной оплаты.
- [ ] Сделка закрывается успешно только после выполнения расчетного сценария.

## 8. Отчеты

- [ ] Есть отчет по количеству сделок.
- [ ] Есть отчет по источникам.
- [ ] Есть отчет по конверсии из дефектовки в ремонт.
- [ ] Есть отчет по отказам и причинам.
- [ ] Есть отчет по СТОА и мастерам.
- [ ] Есть отчет по начисленной комиссии.
- [ ] Есть отчет по оплаченной комиссии.
- [ ] Есть отчет или фильтр по просроченным оплатам.

## 9. Пилотная проверка

- [ ] Проведена тестовая сделка с результатом "отказался".
- [ ] Проведена тестовая сделка с результатом "думает".
- [ ] Проведена тестовая сделка с результатом "согласился на ремонт".
- [ ] Проведена тестовая сделка до стадии "Проверка качества".
- [ ] Проведена тестовая сделка до стадии "Расчеты".
- [ ] Проверены уведомления на тестовых участниках.
- [ ] Проверены права доступа внешнего мастера.
- [ ] Замечания пилота внесены в ТЗ или план доработок.
