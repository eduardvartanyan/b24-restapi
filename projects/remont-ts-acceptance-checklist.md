# Проект "Ремонт ТС": чеклист приемки

Дата создания: 2026-06-17
Дата актуализации: 2026-06-29

Этот чеклист используется для проверки, что настройка CRM-воронки в Битрикс24 готова к работе по текущей версии ТЗ.

## 1. Воронка и карточка сделки

- [ ] Создана отдельная воронка "Ремонт ТС".
- [ ] Созданы все согласованные стадии.
- [ ] Созданы обязательные поля клиента и ТС.
- [ ] В карточке сделки используется контакт клиента.
- [ ] Созданы поля СТОА, мастера СТОА, даты и времени дефектовки.
- [ ] СТОА в сделке связана со смарт-процессом "СТОА".
- [ ] Мастер СТОА в сделке связан с ресурсом бронирования.
- [ ] Созданы поля результата дефектовки.
- [ ] Созданы поля контроля ремонта.
- [ ] Созданы поля проверки качества.
- [ ] Созданы поля расчетов и комиссии.
- [ ] Обязательность полей настроена по стадиям.

## 2. Смарт-процесс "СТОА"

- [ ] Создан смарт-процесс "СТОА".
- [ ] В карточке СТОА хранится название.
- [ ] В карточке СТОА хранится адрес.
- [ ] В карточке СТОА хранятся телефоны.
- [ ] В карточке СТОА хранится график работы.
- [ ] В карточке СТОА хранится список мастеров.
- [ ] В карточке СТОА хранится информация по комиссии.
- [ ] В тестовом контуре заведены 1-2 СТОА.

## 3. Онлайн-запись и мастера

- [ ] Проверена доступность онлайн-записи на текущем тарифе.
- [ ] Настроена тестовая услуга "Дефектовка ТС".
- [ ] Мастера настроены как ресурсы бронирования или специалисты.
- [ ] У каждого мастера есть видимая занятость.
- [ ] Менеджер видит доступные окна мастеров.
- [ ] Менеджер может записать клиента вручную.
- [ ] В сделке фиксируются СТОА, мастер, дата и время дефектовки.
- [ ] Запись не создает конфликтов и двойных бронирований.
- [ ] Клиент получает подтверждение записи.
- [ ] Клиент получает напоминание о визите.

## 4. Задачи мастеру и результат дефектовки

- [ ] После записи на дефектовку создается задача мастеру или ответственному.
- [ ] Текст задачи содержит марку, модель, госномер, клиента, телефон и варианты результата.
- [ ] Мастер получает уведомление о новой записи.
- [ ] Мастер получает ссылку на автоматически созданную задачу.
- [ ] Мастер может оставить комментарий к задаче.
- [ ] Результат дефектовки переносится в поле сделки вручную или автоматизацией.
- [ ] По результату "отказался" сделка уходит в "Отказ / неуспешно".
- [ ] По результату "не приехал" ответственному ставится задача связаться с клиентом.
- [ ] По результату "думает" сделка уходит в "Клиент думает".
- [ ] По результату "согласился на ремонт" сделка уходит в "Контроль начала ремонта".

## 5. Контроль ремонта и проверка качества

- [ ] На стадии "Клиент думает" создается задача повторного контакта.
- [ ] На стадии "Контроль начала ремонта" создается задача ответственному.
- [ ] Ответственный может заполнить плановую и фактическую дату начала ремонта.
- [ ] На стадии "Ремонт в работе" создается задача контроля окончания.
- [ ] Ответственный может заполнить плановую и фактическую дату окончания ремонта.
- [ ] На стадии "Контроль окончания ремонта" ответственный указывает эксперта.
- [ ] На стадии "Проверка качества" создается задача эксперту.
- [ ] Эксперт фиксирует результат проверки качества в комментарии к задаче.
- [ ] Результат проверки качества переносится в сделку.
- [ ] Ответственный сообщает клиенту о готовности ТС.

## 6. Права доступа

- [ ] Менеджер Форсайта видит и редактирует сделки своей воронки.
- [ ] Руководитель видит все сделки направления.
- [ ] Эксперт видит задачи и сделки, где он назначен.
- [ ] Мастер СТОА не является полноценным пользователем CRM.
- [ ] Мастер СТОА видит только свои записи/задачи и минимальные данные для дефектовки.
- [ ] Мастер СТОА не получает лишний доступ к CRM.
- [ ] Мастер СТОА может оставить комментарий к задаче.

## 7. Уведомления

- [ ] Клиент получает подтверждение записи на дефектовку.
- [ ] Клиент получает напоминание о визите.
- [ ] Мастер получает уведомление о новой записи.
- [ ] Мастер получает напоминание указать результат дефектовки.
- [ ] Ответственный получает задачу проверить результат дефектовки.
- [ ] Ответственный получает задачи по началу и окончанию ремонта.
- [ ] Эксперт получает задачу по проверке качества ремонта.

## 8. Расчеты

- [ ] Создан товар "Комиссия за ремонт".
- [ ] Ответственный может добавить товар в сделку.
- [ ] В сделке фиксируется сумма комиссии.
- [ ] Ответственный может выставить счет СТОА.
- [ ] В сделке фиксируется статус оплаты от СТОА.
- [ ] После оплаты сделка переводится в "Успешно завершено".

## 9. Пилотная проверка

- [ ] Проведена тестовая сделка с результатом "отказался".
- [ ] Проведена тестовая сделка с результатом "не приехал".
- [ ] Проведена тестовая сделка с результатом "думает".
- [ ] Проведена тестовая сделка с результатом "согласился на ремонт".
- [ ] Проведена тестовая сделка до стадии "Проверка качества".
- [ ] Проведена тестовая сделка до стадии "Расчеты".
- [ ] Проверены уведомления на тестовых участниках.
- [ ] Проверены права доступа мастера СТОА.
- [ ] Замечания пилота внесены в ТЗ или план доработок.
